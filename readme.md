## 1.前言
```
*我写EasyLinker已经快两年了，往事历历在目，记得还是2017年的春节有了这个想法，后来在2017年7月推出了V1.0,可以说是很烂的一个版本，什么都不支持，仅仅实现了MQTT消息发送，那时候还停留在大学生的水平，无奈之下只能重新开始计划写2.0.
那时候是8月，我找到了第一份实习工作，但是我走进哪家公司的第一天我就被里面死气沉沉的气氛给整的浑身不自在。每个人都在自己的座位上如同机器人一样坐着也不说话。
*第一周，让我们实习生用Spring3.1这个框架，还有Struts1。当时我还专门把JDK从 8降低到了6。公司的技术栈很落后，基本上都是十年前的技术，当时就很好奇，这么老的技术，公司怎么发展成上市公司的？到如今才懂，技术先进与否和公司赚钱没关系，
要说关系最大的还是自己的发展吧。
*于是我第一周周末就果断的辞职了，当时辞职完了以后，靠着女朋友的微薄工资资助我，才没有被饿死在繁华的科技园街头。（我非常感谢我的傻傻的女朋友一路对我的支持，最艰难的时候，当时她的工资3000都不到，还要租房，还有支持我 ，同意我辞职回去深造，那一段时间她又瘦又黑，经常哭，但是还是没有放弃我，我非常感谢她）。
我辞职以后就回到了学校的实验室，开始艰苦的的深造学习，买了Java的高级教程，还有C++ 的一些教程，恶补前两年没有学到的知识，从电子电路到操作系统到网络，直到上层的HTML，我在哪个暑假愣是吧自己关在小房间过了一遍。
*大四开学以后比较轻松（三流本科，没多少课），还是继续学习，时间大把的在实验室，晚上几乎都是十二点一点回宿舍，那一段时间，我突然有一天发现头发两个月没剪了。V2.0就是我在12月开始写的，积累了物联网大量的知识以后，开始写V2，实施证明经过时间打磨的东西总是好的，V2果然也吸引了一批玩家，还有很多付费客户，让我几个月的付出有了回报。从18年春节开始以后，我就不断的利用空闲时间继续开发V2.X，版本，从原来的仅仅支持MQTT协议到后来的支持视频监控，网页回调等很多高级功能，同时一起的团队也有六七个人，我感觉很开心。直到现在还在完善2.0。
*上面讲的是我的一些经历，接下来讲的是我为何停止2.0来开发3.0。
*众所周知，2.0借助来EMQ这个优秀的开源物联网消息Broker，但是EMQ有个缺点就是，使用Erlang开发，我们普通程序员难以驾驭这个语言，导致二次开发很难受，而且这个EMQ仅仅关注MQTT本身，
而不是业务逻辑，所以要想扩展业务逻辑，必须要自己开发，难度不是一般的大。EasyLinkerV2就是这么来的，我封装了繁杂的业务逻辑，和Java结合起来，实现来物联网设备的管理控制，本来我的
设计初衷是"一个轻量级的物联网平台"，但是越到后期越庞大了，很多朋友抱怨玩起来很难，于是我下定决心重新做一个真正的一个jar包到处运行的新版本。
新版本是我看了Netty，NIO等这些新技术以后准备采取的方案，也是对我技术的一个全新挑战。
希望3.0能给大家带来"不一样的滋味"。
------------------------------------------By wwhai@2018-9-8
```
